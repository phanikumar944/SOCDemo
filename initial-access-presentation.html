Part A

Defensive Reconnaissance

Before any attack, the adversary studies our perimeter from the outside — exactly the view a stranger on the internet gets of us.

Step 1 · Service Identification

What doors are open?

A service scan reveals which common entry points are exposed — Remote Desktop (RDP), and web services (HTTP/HTTPS). If RDP or an IIS web server answers from the internet, the attacker has found a way in.

Step 2 · RDP Security Check

How strong is the lock?

The attacker probes RDP's encryption level. "High" suggests Network Level Authentication (NLA) is on, which slows brute-force. "Low" or "Client Compatible" signals a soft target — a prime candidate for Initial Access Brokers.

Part B · Offense

RDP Brute Force — The IAB Vector

Initial Access Brokers don't guess thousands of usernames. They lock onto one high-value account — usually Administrator — and hammer it with common passwords until something breaks.

1

Build a focused wordlist

A short list of the most common leaked passwords — the ones humans actually reuse.

2

Target a single account, slowly

One username (Administrator), one attempt at a time. Going slow is deliberate — it stays under account-lockout thresholds that would otherwise stop the attack in the real world.

★

The fingerprint

Many password attempts, but always against the same single user. Remember this pattern — the Blue Team will use it.

Part C · Offense

Credential Stuffing — The Infostealer Vector

Infostealers like RedLine harvest millions of username:password pairs from infected browsers. Attackers buy these dumps off the dark web and spray them at login portals (RDWeb, VPN).

1

Load a stolen combo list

A database dump of real credentials — each line is a different user with a different password, unlike the single-user brute force.

2

Spray across many accounts

Rather than one user, the attacker tests dozens of accounts at once, looking for any pair that still works.

★

The fingerprint

Many failed logins, but spread across many distinct usernames. This is the opposite signature of Part B.

Two attacks, one equation

IAB vs. Infostealer

Both flood the logs with failures. The difference is invisible to a firewall but obvious to math: count the distinct usernames.

Brute Force (IAB)

Part B
  • Targets one account
  • Many passwords, single username
  • Low distinct-username count
  • Goal: break a known admin

Credential Stuffing

Part C
  • Targets many accounts
  • Each login a new username
  • High distinct-username count
  • Goal: find any valid pair

The QRadar tie-in: a single query on COUNT(DISTINCT username) tells the two apart instantly.

Module 3 · Offense

Phishing & Command-and-Control

A single click on a malicious document delivers a hidden backdoor. This is how most real breaches actually begin.

Step 1

Stand up C2

The attacker launches a Command-and-Control server on Kali — a fake web service the victim will secretly call home to.

Step 2

Weaponize a document

A malicious Office macro is generated, designed to evade defenses (AMSI bypass) and dial back to the C2 channel.

Step 3

"Enable Content"

The victim opens the invoice and enables macros. An agent checks in — the attacker now has a hidden foothold inside the network.

"The user opened the invoice, clicked enable macros — and we have a backdoor. No exploit, no malware alert. Just a click."

Module 1 · The Crown Jewel

The Web Shell Pivot

A single web-app bug bypasses every perimeter defense and chains all the way to Domain Controller compromise — using nothing but legitimate, inherited credentials.

Bypasses the firewall

Traffic rides over normal web ports (80/443) that must stay open — so the perimeter never blinks.

Bypasses antivirus

The attacker uses built-in, trusted tools instead of malware, so signature-based AV stays silent.

Inherits identity

The web server hands over its service account. No password was ever cracked.

Module 1 · The Kill Chain

From Upload to Domain Admin

Six moves take the attacker from an unauthenticated web form to full control of the Domain Controller.

01

Upload web shell

Abuse a file-upload flaw on the HR portal, disguised as a report.

02

Run recon

Discover the inherited service account and enumerate every domain user.

03

Drop a quieter shell

Trade the noisy web shell for a stealthier reverse shell.

04

Catch the callback

The reverse shell connects back — a fully interactive session on the web server.

05

Pivot with WMI

Reuse the inherited credentials to log into the Domain Controller remotely.

06

Own the domain

A prompt on the DC. The network belongs to the attacker.

How the SOC Catches It

The firewall and AV missed it because nothing looked "malicious." Detection comes from behavior — watching what trusted accounts actually do.

Wazuh · Rule 110002

Web shell calling home

A web server process reaching out to download a payload is highly abnormal — Wazuh flags the C2 download.

Wazuh · Rule 110003

WMI lateral movement

A service account suddenly authenticating to the Domain Controller via WMI is the signature of a pivot.

QRadar · The Math Query

One Equation, Two Verdicts

Both attacks generate a wall of failed logins. QRadar groups the failures by source and asks a single question.

Few distinct usernames

Verdict
  • Many failures, one or two accounts
  • Signature of Brute Force
  • → Initial Access Broker (Part B)

Many distinct usernames

Verdict
  • Many failures, many accounts
  • Signature of Credential Stuffing
  • → Infostealer spray (Part C)

COUNT(DISTINCT username) — the single number that tells the analyst which attack is underway.

Wrapping up

Key Takeaways

Identity is the new perimeter

Every vector — RDP, stuffing, phishing, web shell — succeeded by abusing valid credentials, not by breaking through walls.

Prevention will fail

AV and firewalls stayed silent the whole time. Assume breach and invest in detection.

Behavior beats signatures

Wazuh caught the pivot by spotting abnormal actions, not known-bad files.

Context turns logs into answers

One smart QRadar query distinguished two attacks that looked identical to the firewall.
